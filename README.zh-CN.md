# 论大语言模型之理解

**An Essay Concerning LLM Understanding 的核心定位是一个模型无关的多层级向量存储系统。** 它同时实验多层语义记忆、跨层映射和程序性小径。标题是对 John Locke 的 *An Essay Concerning Human Understanding* 的致意。

[English README](README.md) · [研究数据](research/) · [规模实验方案](research/SCALING_STUDY.md)

> 当前是研究型 Alpha。核心结构可以运行，但现有小规模实验没有证明多层检索普遍比单层向量检索更快或更准确。本库的目的，是让这个问题可以被复现和继续检验。

## 它做什么

这是一个实验性的**多层级向量存储**：系统不把原文、人类解释、AI抽象、批判和程序性经验全部压进一个无差别的向量层，而是将向量及其语义元素组织为彼此平行、可单独选择、可比较、可追踪和可显式映射的知识层。

“多层级”指知识与向量在逻辑上的分层；底层实现既可以在同一个物理 collection 中使用层级元数据，也可以使用多个物理 collection。核心不是容器数量，而是层级之间保持区别并允许元素级映射。它支持：

- 无永久根层的知识结构；
- 来源可追踪的语义节点；
- 支持、矛盾、限定、解释、派生、类比等显式关系；
- 受深度、宽度、关系类型、循环检测和信息增益约束的联想；
- 平行于知识层的“小径层”；
- 查询时先查小径，未命中才自由探索；
- 查询成功后产生候选小径，多次成功才晋升；
- 不配置大模型也能返回证据图；
- 可替换的生成模型、嵌入模型、向量库和文档解析器。

## 快速开始

需要 Python 3.11+。仓库不包含任何大模型、嵌入模型、Qdrant服务或预制知识库。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .
essay-understanding serve
```

打开 `http://127.0.0.1:8765/docs`。默认使用演示性质的确定性哈希嵌入器和 SQLite，能够运行证据检索，但不生成自然语言答案。

若要使用自己的模型：

```env
AEC_LLM_BASE_URL=http://127.0.0.1:1234/v1
AEC_LLM_MODEL=your-model
AEC_LLM_API_KEY=

AEC_EMBEDDING_PROVIDER=sentence_transformers
AEC_EMBEDDING_MODEL=your-embedding-model
AEC_VECTOR_STORE=qdrant
AEC_QDRANT_PATH=./data/qdrant
```

本项目不要求 Qwen、Ollama 或 BGE-M3；用户可以接入本地或在线的兼容接口。

## 一种关于理解的工作假说

本项目从一个有限的判断出发：学习经常表现为新材料与既有结构之间建立可修正的对应关系。映射不是同一；有意义的映射必须保留差异、条件和失效边界。矛盾也不只是检索错误，它可能暴露错误映射、遗漏条件、来源冲突或需要修改的概念边界。

因此系统区分：

- 原文与对原文的理解；
- AI自己形成的抽象与人类教授的解释；
- 语义相似与逻辑支持；
- 命题性记忆与“过去怎样成功找到答案”的程序性记忆；
- 暂定、强化、修正和退役的历史状态。

这是一种可实现、可检验的记忆假说，不是完整的人类学习理论。

## 信息处理链路

1. 文本进入系统并保存来源、哈希和位置。
2. 原文、解释或抽象成为平行知识层，而非永久根层的附属物。
3. 文本被清洗、分段，同时保留来源指针。
4. 可替换的嵌入器建立候选检索索引；向量不是知识的唯一副本。
5. 人类解释作为外部教授的知识进入；模型抽象作为自生成知识进入。
6. 向量相似度只提出跨层比较候选，不直接判断支持或矛盾。
7. 关系判断保存方向、条件、置信度、证据和来源。
8. 用户提问时首先检索小径层。
9. 成熟小径命中后限定起始层、关系、深度、宽度与停止条件；未命中则自由探索。
10. 联想深度是最大研究预算；循环检测与信息增益允许提前停止。
11. 证据被限制在预算内；未配置模型时直接返回证据图，配置模型时强制引用已有节点。
12. 一次成功的自由探索形成候选小径；相似路径重复成功后才晋升为成熟小径。
13. 所有查询路径和小径使用历史可审计；知识可导出，向量索引可以重建。

这些规则全部存在于本项目的代码、数据库结构和验证器中，不存在于某个特定模型的隐藏 Skill 或提示词里。

## 当前实验结果

早期本地原型使用 BGE-M3、Qwen3 14B、SQLite和嵌入式Qdrant。10个不同类别问题分别运行单层、多层、多层加程序性技能，共30次查询：

| 指标 | 单层 | 多层 | 多层＋程序性技能 |
|---|---:|---:|---:|
| 平均总延迟 | 5.908秒 | 8.386秒 | 6.757秒 |
| 标准答案相似度 | 0.805810 | 0.805606 | 0.806572 |
| 论点覆盖率 | 0.642034 | 0.641771 | 0.650433 |
| 引用—证据对齐 | 0.770266 | 0.795861 | 0.844410 |

纯多层慢约44%，没有显示明显准确度优势。程序性技能降低了回答生成时间并提高引用对齐，但当时它在完整多层检索之后才运行，端到端仍比单层慢。当前开源版已经改为“小径先查”，但尚未用相同模型完成新一轮正式对照，因此不能把旧数据当作新顺序已经更快的证据。

逐案例数据和稳定性观察位于 [`research/results`](research/results/)。

## 为什么仍值得研究

如果它最终在普通问答速度上没有优势，多层结构仍可能在来源追踪、保存分歧、表达矛盾、知识修正历史、程序性记忆和可审计路径方面具有价值。这些方向可以用于研究长期记忆、元认知和自我修正，但不能据此声称系统拥有意识或主观体验。

## 公开的规模实验邀请

受本地硬件成本和模型推理时间限制，目前只能进行小规模实验。我们希望知道，当节点从数千增加到数百万、小径从少量增加到大量时：

- 单层、多层和小径优先的速度曲线如何变化；
- 小径经过预热后能否偿还学习成本；
- 更深联想是否提高召回，还是只增加噪声；
- 小径层变大后误命中率如何变化；
- 哪一个规模区间中，多层结构的额外复杂度才真正值得。

有硬件条件的研究者可以按照[规模实验协议](research/SCALING_STUDY.md)运行并提交原始数据。负结果和没有优势的结果同样欢迎。
